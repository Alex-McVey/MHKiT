Collaboration¶

MHKiT-Python welcomes feedback and code contributions.

Software developers, within the core development team and external collaborators, are expected to follow standard practices to document and test new code. Software developers interested in contributing to the project are encouraged to create a Fork of the repository and submit a Merge Request using GitLab. The merge requests will be reviewed by the core development team.

Cotinuous Integration¶

MHKiT-Python includes continuous integration software tests and documentation builds that are run using a GitLab CI pipeline. The tests are run and documentation is rebuilt each time changes are made to the repository. The tests are designed to ensure that the code is performing as expected. New tests and documentation are developed each time new functions are added or modified. Testing status (passing/failed) and code coverage statistics are posted on the README page. Results from the software tests and documentation builds can be viewed at Pipeline page. Tests can also be run locally using the Python package nose, and documentation can be build locally using the Python package Sphinx.
